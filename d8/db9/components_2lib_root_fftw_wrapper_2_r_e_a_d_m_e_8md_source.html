17 Making and Installing
18 ======================
19 i) Ensure both ROOT and FFTW3 are on your system and that ROOTSYS is correctly set (or root-config is in your path) and that the directory containing libfftw3 is in either the system or your own LD_LIBRARY_PATH.
20 ii) Make sure that cmake is installed on your system
21 iii) Set the environmental variable ANITA_UTIL_INSTALL_DIR to point to your desired installation dir (libraries will be installed in a lib subdirectory from this location)
22 iii) To build the code:
23 
24 
25  Right now, we recommend you build using cmake:
26 
27  (optional) make configure
28  make
29  make install
30 
31  To use the old build system:
32  (optional) modify Makefile.config to suit your needs
33  make legacy
34  make legacy-install
35 
36  Trying to mix the two build systems might get things confused

52 Usage from within ROOT
53 =======================
54 a) Ensure that the directory containing the libRootFftwWrapper library is in your LD_LIBRARY_PATH (e.g. ANITA_UTIL_INSTALL_DIR/lib )
55 b) To load the library you can (note the lack of .so or .dylib):
56 gSystem->Load("libRootFftwWrapper");
57 c) The FFTWComplex and FFTtools classes should now be accessible from the ROOT prompt or ROOT macro.

64 Compatibility Mode (cmake only)
65 ===========================
66 
67 At some point, FFTtools changed from a static-method-only class to a namespace
68 so that other things could be added to the FFTtools namespace. This was
69 thought not to adversely affect anyone, since the FFTtools::doFFT, etc. usage
70 would remain the same. However, apparently some users had code which did things
71 like: FFTtools tools; tools.doFFT( );. Even though the instance is extraneous,
72 it may be a pain to go through a lot of legacy code to fix it. For this reason,
73 a libRootFftwWrapperCompat shared library is also created, which should be compatible with
74 old uses of the library. To compile old code against it, you will need to define FFTTOOLS_COMPAT_MODE somehow, either like:
75 
76 #define FFTTOOLS_COMPAT_MODE
77 #include "FFTtools.h"
78 
79 or by adding -DFFTTOOLS_COMPAT_MODE to your compilation flags (either with CXX_FLAGS or in the build system)
